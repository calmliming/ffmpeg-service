# FFmpeg Service 部署指南

> 适用环境：1Panel 面板 + Docker Compose

## 前置要求

- 服务器已安装 Docker（1Panel 默认自带）
- 推荐配置：2核2G 及以上

---

## 一、本地打包

在本地项目根目录执行：

```bash
tar --exclude='./node_modules' \
    --exclude='./dist' \
    --exclude='./output' \
    --exclude='./uploads' \
    --exclude='./*.tar.gz' \
    -czf ../ffmpeg-service-deploy.tar.gz .
```

打包文件生成在项目上一级目录：`ffmpeg-service-deploy.tar.gz`

---

## 二、上传到服务器

1. 打开 1Panel → **文件**
2. 进入 `/opt` 目录，新建文件夹 `ffmpeg-service`
3. 进入该文件夹，点击**上传**，选择 `ffmpeg-service-deploy.tar.gz`
4. 上传完成后右键该文件 → **解压** → 解压到当前目录

---

## 三、启动服务

打开 1Panel 顶部导航 → **终端**，执行以下命令：

```bash
cd /opt/ffmpeg-service
docker compose up -d
```

> 首次启动需构建镜像，耗时约 3~8 分钟，请耐心等待。

查看启动日志：

```bash
docker compose logs -f
```

看到以下输出说明启动成功，按 `Ctrl+C` 退出日志：

```
FFmpeg Service v1.1.0 已启动: http://localhost:9527
```

---

## 四、开放防火墙端口

1Panel → **安全** → **防火墙** → **添加规则**

| 字段 | 值 |
|------|----|
| 协议 | TCP |
| 端口 | 9527 |
| 来源 | 0.0.0.0/0 |

---

## 五、验证部署

浏览器访问：

```
http://你的服务器IP:9527/api/health
```

返回以下内容即部署成功：

```json
{ "success": true }
```

---

## 常用运维命令

```bash
# 查看服务状态
docker compose ps

# 查看实时日志
docker compose logs -f

# 重启服务
docker compose restart

# 停止服务
docker compose down

# 更新部署（重新上传解压后执行）
docker compose down
docker compose up -d --build
```

---

## 环境变量说明

配置文件：`/opt/ffmpeg-service/.env`

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `9527` | 服务端口 |
| `FFMPEG_PATH` | `ffmpeg` | FFmpeg 可执行路径 |
| `FFPROBE_PATH` | `ffprobe` | FFprobe 可执行路径 |
| `SAVE_OUTPUT_FILE` | `true` | 是否保留输出文件 |
| `MAX_CONCURRENT_TASKS` | `2` | 最大并发任务数（2核2G 建议设为 2） |
| `REDIS_URL` | `redis://redis:6379` | Redis 连接地址 |
